算法思路

笛卡尔树，顾名思义，和笛卡尔没有任何关系。笛卡尔树是用来优化 treap 的建树的，不支持修改。

笛卡尔树的每个节点都有一个 key 和一个 value ，其中 key 满足 BST 性质，value 满足堆性质。

首先，我们对节点按照 key 升序排序。

然后，维护一个 value 升序的单调栈。

对于第 $i$ 个节点，它是它在单调栈中压住的那一个节点的右儿子，它的左儿子是最后一个被弹出的节点。

比如说：

key: 1 2 3 4 5 6 7 8
value:3 4 5 2 6 1 8 7

算法应用

优化 BST

这时候有人就会说了：“啊~，优化 BST 的方法那么多，各种平衡树都是干这个事的，为什么还要搞一个静态的笛卡尔树呢？”

说对了一半。平衡树是可以优化 BST，但是它们都是基于旋转/分裂的，一旋转/分裂，它的顺序就变了。虽然还是满足 BST 性质，但是它已经不是本来按部就班的 BST 的结构了。

这时候有人就会说了：“啊~，为什么要管那么多呢？满足 BST 性质就够了呀！”

那请你用平衡树做一下这道题。

我们想一下如何优化。

BST 之所以和其他平衡树的形态不同，是因为平衡树在插入顺序角度来看完全无序，而 BST 中的父结点一定先于子结点插入。

既然如此，其实就是 BST 的插入顺序满足堆性质，实际值满足 BST 性质，就是一个笛卡尔树。

$10^7$ RMQ

假设有这样一道题。

有一个长度为 $n$ 的数组和 $q$ 个查询，每次查询都询问某一区间的最小值（即 RMQ ）。

输入的数组和查询都由种子生成，输出所有答案的异或和（即不用考虑 IO ）。

其中， $1 \leq n,q \leq 10^7$。

显然，这次连预处理都不能带 $\log$ 了， st 表不能用。

考虑如何用笛卡尔树解决。

如果我们用数组下标当 key ，元素值当 value ，我们会发现，一个节点的左子树的下标都比自己小，元素值都比自己大，右子树的下标都比自己大，元素值也比自己大。

于是，我们可以得到，从 $l$ 到 $r$ 的最小值是它们的 最近公共祖先 的 value 。

而最近公共祖先可以用 Tarjan 离线 $O(n)$ 全部处理，所以总时间复杂度为 $O(n)+O(q)+O(n) = O(n+q)$ 。
